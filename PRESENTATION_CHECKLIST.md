# Presentation Submission Checklist

Required (submit all):
- Presentation slides (final deck, ~10 slides)
- 10-minute recorded video (YouTube unlisted or Canvas upload)
- Code repository (public GitHub link) OR zipped folder uploaded to Canvas
- Well-documented code: clear `README.md`, usage instructions, and comments for non-trivial logic

Suggested slide order (~10 slides):
1. Title slide — project title, authors, course, date
2. Problem definition — what you solved and why it matters
3. Dataset — YouCook2, key stats, subset used
4. Approach — overall architecture diagram (encoder, ASR, T5 decoder)
5. Model / Methods — model choices and training setup
6. Implementation details — important scripts, checkpoints, environment
7. Evaluation — metrics, table of results, example outputs
8. Demo / Visuals — qualitative captions and timestamps
9. Challenges & limitations — what failed and why
10. Future work & conclusion

Submission tips
- Put the public GitHub link in the slide deck and README.
- Add exact run commands to the README.
- Verify that uploaded video is accessible (unlisted link) before deadline.
- Presentation video link: https://youtu.be/LiwOHimTcq8?si=hH66QVTxH7oDvIJg
